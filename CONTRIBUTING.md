# Contributing

这个仓库不是传统开源项目那套“欢迎所有人随便提一堆 PR”。

它更像一个**可迁移的班主任 skill 仓库**。
贡献方式也应该现实一点：

## 你适合贡献什么

欢迎补这些：
- 更通用的班主任模板
- 可复用的脱敏示例数据
- 能独立运行的小脚本
- 更清楚的 skill 文档
- 适合 agent 自动安装/加载的目录结构改进
- 对接飞书 / 企业微信 / 钉钉 / cron 的非私密骨架代码

## 不要提交什么

绝对不要提交：
- 真实学生成绩
- 真实姓名、电话、家庭信息
- 家校沟通原文
- 教师内部敏感记录
- token、cookie、sessionKey、平台密钥
- 任何本地私有 `data/` 内容

## 提交前最低要求

1. 保持公开版本可发布
2. 不要引入真实隐私数据
3. 脚本至少能本地运行
4. 文档要说人话，别写成空话
5. 不要把路线图吹成已经完成的功能

## 目录约定

- `skills/**/SKILL.md`：skill 主文档
- `skills/**/README.md`：对外使用说明
- `skills/**/references/`：可公开模板与参考材料
- `skills/**/scripts/`：脚本
- `skills/**/examples/`：脱敏示例
- `skills/**/data/`：本地私有，默认不进仓库

## 对贡献者最重要的一条

如果你是老师，最好的贡献往往不是“写更复杂的代码”，而是把你真实工作里的高频场景，整理成：
- 一个稳定任务描述
- 一个可复用模板
- 一个清晰输入输出格式

这样别人的 agent 才真能接得住。

## 推荐贡献方向

优先欢迎这几类：
- 家长通知模板标准化
- 学生评语结构化输入模板
- 班务提醒 / 月报 / 日报模板
- 成绩分析报告格式改进
- 家校沟通语气策略沉淀
- 多平台接入骨架

## Pull Request 风格

请尽量做到：
- 一次只改一类东西
- 标题写清楚改了什么
- 如果改了 skill 行为，顺手改文档
- 如果加了脚本，给一个最小示例

## 一句话原则

这个仓库追求的不是“更炫”，而是：

> 让别的老师把仓库链接发给自己的 AI，它就能尽快装上、看懂、开始干活。
