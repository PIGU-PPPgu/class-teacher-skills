# 班主任助理

这不是单纯的“文案生成器”。

它现在是一个 **班主任工作助手 + 模板与脚本工具包**，下一步要长成一个 **会记、会催、会生成、会等你确认再发送** 的主动工作流系统。

---

## 1. 当前已经能做的事

### 1.1 沟通文案生成

**家长通知**
```text
帮我起草一个关于[主题]的家长群通知
- 考虑语气：[正式/亲切/紧急]
- 包含要点：[要点列表]
```

**学生评语 / 寄语**
```text
为[学生姓名]写一段期末评语
- 特点：[性格/表现特点]
- 建议：[需要提及的改进点]
```

**班会发言稿**
```text
准备一次关于[主题]的班会发言
- 时长：[几分钟]
- 目标：[教育目标]
```

### 1.2 数据分析与报告

**成绩分析**
使用 `scripts/analyze_grades.py`：
```bash
python scripts/analyze_grades.py --file 成绩表.xlsx --output 分析报告.md
```

当前能力：
- 班级整体趋势分析
- 各科均分 / 极值 / 中位数 / 标准差
- 分数段分布统计
- 低分学生提示
- 多科时的简单偏科识别

**考勤统计**
使用 `scripts/attendance_report.py`：
```bash
python scripts/attendance_report.py --file 考勤数据.csv --month 2026-02
```

当前能力：
- 按学生汇总考勤状态
- 输出整体考勤统计
- 快速生成月度考勤报告草稿

### 1.3 学生成长档案

**谈话记录模板**
见 `references/student_record_template.md`

记录要素：
- 时间、地点、参与人
- 谈话背景
- 学生状态描述
- 关键对话内容
- 后续跟进计划

**特殊学生跟踪**
使用 `scripts/student_tracker.py` 管理：
```bash
python scripts/student_tracker.py add 学生姓名 --issue 问题类型 --desc 情况描述
python scripts/student_tracker.py update 学生姓名 --note 新进展 --type 谈话记录
python scripts/student_tracker.py report 学生姓名
python scripts/student_tracker.py list
```

当前能力：
- 建立学生跟踪档案
- 持续追加跟踪记录
- 输出单个学生成长跟踪报告
- 标记状态变化

### 1.4 日程与提醒模板

**重要日期跟踪**
见 `references/class_calendar.md` 模板

当前覆盖：
- 考试时间表
- 家长会安排
- 缴费截止日期
- 学生生日清单
- 学校活动日程

### 1.5 教学支持

当前已整理方向：
- 教学视频转录和摘要
- 网页资料提取和整理
- 练习题生成思路
- 小组分配与课堂活动设计

## 3. 主动工作流路线图（vNext）

下面这些场景，是这个 skill 最有价值的下一步，不该继续散在聊天记录里。

### 3.1 定时推送类
- 每周任务清单自动推送
- 每日 7:00 今日提醒
- 考试前 N 天复习提醒
- 假期作业打卡提醒
- 校历事件提前提醒
- 值日生轮换提醒
- 天气 / 穿搭 / 极端天气提醒

### 3.2 班务材料自动生成类
- 家长群消息润色 + 待确认发送
- 每月班务日志到期提醒 + 初稿生成
- 班会课主题建议 + 教案初稿
- 家长会全流程材料
- 运动会 / 春游 / 研学全套材料自动备菜
- 新学期开学清单自动启动
- 考试全流程管家

### 3.3 学生管理 / 家校沟通类
- 评语平时采集、期末批量生成
- 学生异常主动预警（迟到、违纪、波动）
- 作业未交自动追踪 + 分层提醒
- 请假自动流转
- 任课老师协调记录与跟进
- 家长群争执预警 + 回复草稿
- 生日 / 特殊日期提醒

### 3.4 分析汇总类
- 每日班级播报 / 班级日报
- 学生画像自动更新
- 班级趋势分析
- 综合素质评价初稿生成
- 班费记账与月底汇总

## 4. 最值得优先闭环的 5 个能力

如果只先做最值钱的，不要铺太大，优先级建议：

1. **家长群消息润色 + 确认发送**
2. **每日 / 每周提醒自动生成**
3. **评语平时采集，期末批量生成**
4. **作业未交 / 迟到异常提醒**
5. **班级日报 / 班务月报自动汇总**

原因很简单：
- 高频
- 痛点清晰
- 价值容易感知
- 最容易接成真正闭环

## 5. 能力边界：哪些是现在就能用，哪些还不能吹太满

### 5.1 现在已经能用
- 自然语言起草通知、评语、发言稿
- 跑成绩分析脚本
- 跑考勤汇总脚本
- 做学生跟踪记录与报告
- 使用模板整理班务材料
- 用脱敏示例数据演示工作流

### 5.2 还在路线图里，不能假装已经打通
- 钉钉 / 微信 / 飞书群自动定时发送
- 家长群争执自动监控与介入
- 接入考勤系统后自动识别连续迟到
- 接作业系统自动拉未交名单
- 请假后自动通知任课老师并闭环
- 每天自动拉天气后定时发群
- 月底自动汇总全月班务日志并生成正式初稿

这些能力要补齐下面几层：
- 平台 API 接入
- 定时任务调度（cron）
- 数据结构定义
- 审核 / 确认节点
- 对外发送执行层

一句话说清：

> 现在已经有“技能骨架 + 模板 + 脚本 + GitHub 可发布结构”；
> 还没有全部接成一个能自动在群里跑的班主任操作系统。

## 6. 推荐使用方式

### 方式 1：在支持 skill 的 agent 环境里直接调用
示例：
- “帮我起草一个关于春游安全事项的家长群通知”
- “分析这次月考成绩，找出进步最大的 5 名学生”
- “记录一下今天和小明的谈话，要写进学生跟踪”
- “给我准备一份下周家长会发言提纲”

### 方式 2：把 skill 当成模板包和脚本工具包用
即使没有完整 agent 平台，也可以：
- 直接阅读 `SKILL.md`
- 使用 `references/` 模板
- 运行 `scripts/` 里的脚本
- 用 `examples/` 理解输入输出格式

### 方式 3：后续接成主动工作流产品
调用链会是：
- 数据源：考勤 / 作业 / 校历 / 天气 / 班费 / 班级记录
- 触发器：定时器、阈值、事件变更
- 决策层：skill 规则 + 模板 + 脚本
- 输出层：生成草稿、提醒班主任、确认后发送

## 9. 数据安全提醒

⚠️ **学生隐私保护**
- 所有真实学生数据仅存储在本地
- 公开仓库只放模板、脚本、说明和脱敏示例
- 生成报告时要注意脱敏处理
- 不要提交联系方式、谈话原文、成绩明细、家庭信息
- 不要提交 token、cookie、sessionKey、平台密钥

## 10. GitHub 发布建议

为了适合公开仓库发布，建议按下面规则维护：

- **不要提交真实学生隐私数据**：姓名、学号、联系方式、谈话原文、家庭信息都应脱敏
- **优先提交模板和示例**：参考资料、伪造样例数据、通用脚本可以公开；真实班级数据不要公开
- **把本地专有流程写清楚**：例如数据输入格式、依赖库、输出文件格式
- **避免写死绝对路径**：公开版本应优先使用相对路径或环境变量
