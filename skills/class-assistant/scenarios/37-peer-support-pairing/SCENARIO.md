# 场景 37｜学生互助结对 / 帮扶对象分配

## 场景目标
根据学生名单、学习层次和班级管理要求，生成清楚、均衡、便于执行的互助结对或帮扶对象分配方案，帮助班主任快速建立学生之间的支持关系与跟进机制。

## 适用时机
- 新学期建立学习互助对子
- 需要安排一帮一帮扶或同伴支持
- 需要兼顾学习层次、责任心、沟通能力、性格稳定性和执行情况
- 需要避开不适合同组或同伴的学生组合
- 需要输出学生执行版和老师内部观察版
- 不适合替代正式心理评估或处分决定

## 输入
- 必填信息：学生名单、结对目标、分配规则
- 可选信息：学业层次、性格特点、责任心、品行、上课态度、学习态度、沟通能力、回避组合、重点关注对象
- 缺失时默认：优先按互补原则输出一版基础结对表

## 输出
- 默认输出：一版互助结对表
- 可选输出：帮扶目标版、阶段跟进版、学生执行版、内部备注版、观察重点摘要
- 输出对象：班主任、学生、家长（如需）

## 推荐语气
- 清楚
- 公平
- 不冗长
- 执行导向
- 便于后续跟进

## 文件说明
- `templates.md`：结对表与说明模板
- `examples/`：脱敏输入输出样例
- `scripts/`：如后续需要可接自动配对脚本

## 直接给 AI 的说法
- 请按这份名单给我出一版学生互助结对安排。
- 请把学习层次、责任心、沟通能力和学习态度一起考虑进去。
- 请给我一版老师内部说明，再给一版可直接对学生公布的版本。

## 注意事项
- 默认先出草稿，发布前建议老师再核对实际关系和班情
- 不要在公布版中公开写学生敏感标签或负面评价
- 若规则冲突，优先标注无法同时满足的条件
