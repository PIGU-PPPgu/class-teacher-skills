# 场景 27｜作业催交跟进表

适合把未交、迟交、补交和催交情况整理成可追踪的执行表。

## 这个场景解决什么问题
- 作业异常学生多，提醒后容易漏
- 不清楚哪些学生已催、哪些已联系家长
- 需要识别重复未交对象
- 需要给周复盘和家校沟通提供依据

## 建议先看
- `SCENARIO.md`：场景说明与边界
- `templates.md`：标准跟进表与重点学生摘要模板
- `examples/input.sample.md`：示例输入
- `examples/output.sample.md`：示例输出

## 典型输出
- 作业催交跟进表
- 重点异常学生名单
- 今日待催交事项

## 一句话用法
把作业名称、学生、未交/迟交状态、催交次数和家长联系情况给 AI，让它整理成催交跟进表。
