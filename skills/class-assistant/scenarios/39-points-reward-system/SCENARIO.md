# 场景 39｜积分量化管理系统

## 场景目标
帮助班主任建立一套可执行、可量化、有激励效果的班级积分管理制度，覆盖加减分规则、游戏化兑换机制、积分收集流程和公示方案，让班级日常行为管理从"凭印象"转为"有数据"。

## 适用时机
- 新学期建立班级管理制度
- 已有基础规则但需要系统化、量化
- 希望引入游戏化机制提升学生参与积极性
- 需要让课代表、值日班长参与日常打分
- 期中或期末需要根据积分做激励、评优或座位调整参考
- 不适合替代学校正式纪律处分或德育评分系统

## 输入
- 必填信息：班级学生名单、希望覆盖的行为维度（如考勤、纪律、学习、卫生等）
- 可选信息：学校已有常规检查项目、班级特色活动、奖惩偏好、是否引入游戏化机制
- 缺失时默认：输出一套包含6大维度的标准积分规则 + 基础收集流程

## 输出
- 默认输出：积分规则表 + 收集流程说明 + 公示模板
- 可选输出：游戏化"班级大富翁"规则、课代表打分授权说明、积分兑换奖励清单、学期积分继承方案
- 输出对象：班主任（内部管理）、学生（规则公布）、家长（可选同步）

## 推荐语气
- 清晰
- 公平
- 正向激励为主
- 规则明确、无歧义
- 便于学生理解和执行

## 文件说明
- `templates.md`：积分规则模板、收集流程模板、游戏化规则模板、公示模板
- `examples/`：脱敏输入输出样例

## 直接给 AI 的说法
- 请帮我设计一套班级积分量化管理方案，覆盖考勤、纪律、学习、卫生、体育、常规六个维度。
- 我有一套加减分规则草稿，请帮我整理成可发布的学生版。
- 请帮我设计一个游戏化的积分兑换系统，类似大富翁的玩法，学生可以用积分参与。
- 请帮我整理课代表和值日班长的积分收集流程，要简单易操作。

## 注意事项
- 默认先出草稿，发布前建议老师根据班级实际情况调整分值和规则
- 所有示例均使用占位符，不出现真实姓名、学校名或班级号
- 游戏化机制建议先小范围试运行再全班推广
- 积分规则公布后需保持稳定性，频繁改动会降低公信力
- 扣分项避免过重，正向激励与负向约束应保持平衡
