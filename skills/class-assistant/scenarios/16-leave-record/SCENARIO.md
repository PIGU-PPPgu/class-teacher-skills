# 场景 16｜请假登记 / 请假说明

## 场景目标
把学生请假信息整理成清楚、可留痕、便于后续跟进的登记稿或说明稿，方便老师统一记录与沟通。

## 适用时机
- 家长请假后需要快速登记
- 需要向科任老师或年级组同步请假情况
- 需要生成一版规范的请假说明
- 需要后续追踪返校、补课或作业安排
- 不适合替代学校正式请假审批系统

## 输入
- 必填信息：学生身份、请假日期、请假原因、请假时长
- 可选信息：是否已请家长说明、返校时间、补作业安排、健康状况备注
- 缺失时默认：只保留已确认事实，不补写未核实原因

## 输出
- 默认输出：一版内部登记 / 说明稿
- 可选输出：家长确认回执版、科任同步版、班主任留档版
- 输出对象：老师内部使用为主，必要时用于对家长确认

## 推荐语气
- 客观
- 简洁
- 规范
- 不夸张

## 文件说明
- `templates.md`：请假登记、同步说明模板
- `examples/`：脱敏输入输出样例
- `scripts/`：如后续需要批量登记脚本，可放这里

## 直接给 AI 的说法
- 帮我把这条请假信息整理成班主任留档版。
- 根据这位学生的请假情况，写一版给科任老师同步的简短说明。
- 把下面信息整理成规范请假登记稿，不要补未确认内容。

## 注意事项
- 默认先生成草稿，不直接外发
- 不写未核实病情细节
- 涉及连续病假、伤情或敏感情况时，保留人工确认环节
