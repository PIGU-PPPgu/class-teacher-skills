# 场景 26｜家校沟通台账

## 场景目标
把与家长的沟通记录整理成可持续维护、可回溯、便于后续跟进的结构化台账。

## 适用时机
- 需要持续记录和家长的沟通情况
- 需要区分已沟通事项、待跟进事项和结果反馈
- 需要为月度复盘、敏感事项留痕或内部交接做准备
- 需要避免“沟通过但找不到记录”
- 不适合替代学校正式系统中的法定留档

## 输入
- 必填信息：日期、学生姓名或编号、沟通对象、沟通事由、沟通结果
- 可选信息：沟通方式、家长反馈、老师判断、后续动作、下次跟进时间
- 缺失时默认：按“时间—对象—事由—结果—后续动作”输出

## 输出
- 默认输出：一版家校沟通台账
- 可选输出：单个学生沟通汇总、待跟进清单、月度沟通复盘版
- 输出对象：班主任、内部协作老师、个人留档

## 推荐语气
- 客观
- 简洁
- 留痕导向
- 可执行
- 不情绪化

## 文件说明
- `templates.md`：台账模板与单次记录模板
- `examples/`：脱敏输入输出样例
- `scripts/`：如后续需要可接 CSV/JSON 维护脚本

## 直接给 AI 的说法
- 请把下面几次家校沟通整理成一份台账。
- 请根据这些零散记录，生成一版待跟进的家校沟通表。
- 请把这名学生最近三次和家长的沟通整理成汇总。

## 注意事项
- 默认用于内部，不直接转发给家长
- 只记录已发生事实，不补写主观推断
- 涉及敏感家庭情况时，控制细节披露范围
