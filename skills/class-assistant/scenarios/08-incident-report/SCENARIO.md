# 场景 08｜突发事件上报 / 情况说明

## 场景目标
帮助老师把突发事件、学生异常情况或处理经过整理成客观、克制、可留档的情况说明，适用于内部上报、后续记录或家校沟通前的底稿准备。

## 适用时机
- 班级内发生突发小事件需要留痕时
- 需要向年级组 / 学校做简要情况说明时
- 需要先形成处理经过底稿，再决定是否对外沟通时

不适合：
- 代替正式调查结论
- 未核实清楚就做责任认定
- 对外公开包含敏感隐私的完整细节

## 输入
### 必填
- 事件时间
- 事件地点
- 事件经过
- 已采取处理措施

### 可选
- 涉及人员
- 当前状态
- 后续处理建议
- 输出对象（内部记录 / 家长沟通底稿 / 学校上报）

### 缺失时默认处理
- 未给输出对象：默认内部记录口径
- 未给当前状态：不强行推断
- 未给后续建议：保留待跟进项

## 输出
默认输出：
- 1 份情况说明 / 事件记录

可选输出：
- 更正式上报版
- 家长沟通底稿版
- 极简留痕版

## 推荐语气
- 客观
- 审慎
- 去情绪化
- 留痕导向

## 文件说明
- `templates.md`：情况说明模板
- `examples/`：脱敏输入输出样例

## 直接给 AI 的说法
- 帮我整理一份今天这件事的情况说明
- 帮我写成给年级组看的内部上报版
- 帮我先做一个家长沟通底稿

## 注意事项
- 只写已知事实与处理动作
- 不提前定责
- 不夸大冲突程度
- 涉及未成年人时尽量最小披露
