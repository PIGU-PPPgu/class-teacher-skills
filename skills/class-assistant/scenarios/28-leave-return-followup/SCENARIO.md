# 场景 28｜请假返校补课跟进表

## 场景目标
把学生请假、返校、补作业、补课和确认状态整理成一份清楚、可执行的跟进表，减少返校后信息断层。

## 适用时机
- 学生请假后需要持续跟进返校与补课
- 需要统一记录补作业、补测、课堂补充安排
- 需要和科任老师或家长同步返校后的执行情况
- 需要避免返校后遗漏关键任务
- 不适合替代学校正式请假审批记录

## 输入
- 必填信息：学生姓名或编号、请假时间、返校时间、请假原因、待补事项
- 可选信息：科任要求、家长沟通情况、完成状态、确认人、备注
- 缺失时默认：按“请假—返校—待补任务—完成状态”输出

## 输出
- 默认输出：一版请假返校补课跟进表
- 可选输出：单个学生返校跟进卡、待完成清单、家校确认版
- 输出对象：班主任、科任老师、内部记录

## 推荐语气
- 客观
- 执行导向
- 清楚
- 便于协作
- 不情绪化

## 文件说明
- `templates.md`：跟进表与返校确认模板
- `examples/`：脱敏输入输出样例
- `scripts/`：如后续需要可接请假台账脚本

## 直接给 AI 的说法
- 请把这几名学生的请假返校情况整理成跟进表。
- 请按待补作业、补课、确认状态给我一版返校清单。
- 请帮我整理哪些学生返校后还有事项没完成。

## 注意事项
- 区分“已返校”和“待返校”状态
- 区分“已补完”和“待确认”状态
- 如需外发给家长，建议另生成家长确认版文案
